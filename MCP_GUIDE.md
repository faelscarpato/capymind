# MCP Guide

Resumo curto: guia de navegação do `capymind` para agentes conectados via MCP ou GitMCP.

## Objetivo
Este repositório foi organizado para responder perguntas sobre:
- projetos do ecossistema Capy
- operação comercial e Mercado Livre
- prompts e fluxos com IA
- documentação técnica e referências
- catálogo de produto e ativos relacionados à Pétala Decor

## Ordem recomendada de leitura
1. [llms.txt](./llms.txt)
2. [README.md](./README.md)
3. [KNOWLEDGE_MAP.md](./KNOWLEDGE_MAP.md)
4. [projects/_featured.md](./projects/_featured.md)
5. índices específicos do domínio consultado

## Como navegar por tipo de pergunta

### Se a pergunta for sobre um projeto específico
1. Procure em [projects/_featured.md](./projects/_featured.md).
2. Se não estiver lá, abra [PROJECT_CATALOG.md](./PROJECT_CATALOG.md).
3. Entre na pasta do projeto em `projects/<slug>/`.
4. Leia primeiro:
   - `overview.md`
   - `purpose.md`
   - `stack.md`
   - `architecture.md`
   - `integrations.md`

### Se a pergunta for sobre operação / negócio
1. Abra [knowledge/business/mercado-livre-operations.md](./knowledge/business/mercado-livre-operations.md).
2. Depois [knowledge/operations/capy-ops-and-ml-workflows.md](./knowledge/operations/capy-ops-and-ml-workflows.md).
3. Use [datasets/_index.md](./datasets/_index.md) para localizar bases descritas.
4. Consulte [references/_index.md](./references/_index.md) para materiais comerciais e técnicos.

### Se a pergunta for sobre prompts ou IA
1. Abra [knowledge/ai/prompt-engineering-notes.md](./knowledge/ai/prompt-engineering-notes.md).
2. Depois [prompts/_index.md](./prompts/_index.md).
3. Consulte [references/technical/prompt-engineering-reference-pack.md](./references/technical/prompt-engineering-reference-pack.md) para apoio.

### Se a pergunta for sobre produto físico, branding ou catálogo
1. Comece por [knowledge/products/petala-decor-catalog-notes.md](./knowledge/products/petala-decor-catalog-notes.md).
2. Depois abra [assets/_index.md](./assets/_index.md).
3. Consulte [knowledge/commercial/marketplace-sales-materials.md](./knowledge/commercial/marketplace-sales-materials.md).

## Regras de interpretação
- Este repositório guarda **conhecimento derivado**, não o código bruto dos projetos.
- Toda leitura de projeto deve considerar que as notas são um resumo orientado por documentação, estrutura e arquivos-chave.
- Quando houver seção **Inferido**, trate como hipótese forte, não como fato absoluto.
- Para rastreabilidade, use `source-path.md` dentro de cada projeto.
